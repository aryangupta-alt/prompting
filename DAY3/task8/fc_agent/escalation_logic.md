# Escalation Logic

## Overview

The customer-service agent includes an escalation mechanism to transfer conversations to a human support representative when the issue cannot be resolved through available tools or when the customer explicitly requests human assistance.

The escalation process is handled through the `escalate_to_human(reason)` tool.

---

## Escalation Triggers

The agent should escalate in the following situations:

### 1. Customer Requests Human Assistance

Examples:

* "I want to speak to a manager."
* "Connect me with a human."
* "I want to talk to customer support."

Reason:
The customer explicitly requests a human representative.

---

### 2. Customer Is Extremely Frustrated

Examples:

* "I'm extremely angry."
* "Nobody is helping me."
* "This is unacceptable."

Reason:
The conversation contains strong negative sentiment and may require personalized support.

---

### 3. Order Disputes

Examples:

* Product not received despite delivery confirmation.
* Fraud or payment disputes.
* Repeated unresolved complaints.

Reason:
These situations often require manual investigation.

---

### 4. Missing Information

Examples:

* Invalid order ID.
* Incomplete customer information.
* Tool returns insufficient data.

Reason:
A human agent may need to gather additional information.

---

### 5. Unsupported Requests

Examples:

* Special refund exceptions.
* Compensation requests.
* Legal or compliance-related concerns.

Reason:
The available tools cannot fully resolve these requests.

---

## Context Passed During Escalation

When escalation occurs, the agent passes:

### Customer Message

The original customer request.

Example:

"I'm extremely angry. Nobody is helping me."

---

### Escalation Reason

A short explanation generated by the model.

Examples:

* Customer requested manager assistance.
* Customer is angry and requires support.
* Order dispute requires manual review.

---

### Tool Response

Example:

```json
{
  "status": "Escalated",
  "message": "A human support agent will contact you shortly.",
  "reason": "customer requested to speak to a manager"
}
```

---

## Escalation Workflow

Customer Query
↓
Agent Analysis
↓
Determine Escalation Need
↓
Call escalate_to_human(reason)
↓
Return Confirmation Message
↓
Human Agent Follow-Up

## Conclusion

The escalation mechanism ensures that emotionally charged, complex, disputed, or unsupported requests are transferred to a human representative. This prevents poor customer experiences and allows the agent to focus on requests that can be resolved using the available tools.
